The Problem

AI agents can think. They can't pay.

Today's AI agents can plan, reason, and execute complex workflows — but when they need to buy compute, data, or API access, they hit a wall built for humans.

🔐

Identity Bottleneck

API keys, accounts, credit cards — all require human identity. An autonomous agent can't sign up for a Stripe account.

💸

Micropayment Failure

Credit card fees make $0.001 transactions impossible. But that's exactly what an AI paying per-API-call needs.

🐌

Speed Mismatch

Agents operate in milliseconds. Traditional payment settlement takes days. The web's payment layer wasn't built for machine speed.

🌍

Global Friction

Banking infrastructure is regional. An agent in Tokyo shouldn't need a US bank account to call an API hosted in Virginia.

The Solution

L402: Payment is authentication

The L402 protocol resurrects HTTP's forgotten 402 "Payment Required" status code, combining it with Lightning Network micropayments and macaroon-based credentials.

1

Agent Requests

AI agent sends HTTP request to an L402-gated API endpoint

2

402 Response

Server returns HTTP 402 with a Lightning invoice + macaroon

3

Lightning Pay

Agent pays the invoice via Lightning (~1 sat, <1 second)

4

Proof of Payment

Agent receives cryptographic preimage as payment proof

5

Access Granted

Agent retries request with proof — resource delivered instantly

Why Lightning wins for AI payments

Traditional payment rails were designed for humans. L402 was designed for machines.

Dimension Traditional L402 + Lightning
Identity Email, phone, credit card None required — payment is auth
Minimum viable payment ~$0.30 (card fees) 1 satoshi (~$0.0009)
Settlement speed Days (chargebacks: weeks) Milliseconds, final
Global access Requires regional banking Permissionless, worldwide
Machine-friendly Requires human in the loop Fully autonomous
Auth model API keys + OAuth + sessions Cryptographic proof of payment
